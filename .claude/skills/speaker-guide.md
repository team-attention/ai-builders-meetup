# Speaker Guide 생성 Skill

밋업 스피커 가이드 문서를 빠뜨림 없이 생성하는 skill입니다.

---

## 사용법

```
/speaker-guide [밋업 폴더명]
```

예: `/speaker-guide 3-100x-engineers`

---

## Phase 1: 초기 정보 수집

스피커 가이드를 만들기 전에 반드시 다음 핵심 정보를 AskUserQuestion으로 수집하세요:

### 필수 질문

1. **밋업 기본 정보**
   - 밋업 이름은 무엇인가요?
   - 날짜와 시작 시간은? (예: 2025년 12월 30일 19:00)
   - 장소는 어디인가요?

2. **발표 형식**
   - 발표 시간은 몇 분인가요? (기본: 15분)
   - 스피커는 몇 명인가요?
   - 패널 토크가 있나요?

3. **타겟 오디언스**
   - 주요 참석자는 누구인가요? (예: AX 리더, 10x 엔지니어)

4. **준비 일정**
   - 호스트 1:1 콜 마감일은?
   - 발표 자료 제출 마감일은?
   - 사전 질문지 전달일은?

---

## Phase 2: 문서 생성

수집한 정보를 바탕으로 speaker-guide.md를 생성합니다.

### 필수 섹션 체크리스트

다음 섹션을 **모두** 포함해야 합니다:

- [ ] **밋업 정보** - 일시, 장소, 형식
- [ ] **타겟 오디언스** - 주요 참석자 설명
- [ ] **준비 일정** - 마감일 테이블
- [ ] **발표 안내**
  - [ ] 시간 (몇 분, 버퍼 시간)
  - [ ] 발표 주제 방향
  - [ ] 발표 자료 제출 방법
  - [ ] 라이브 데모 안내
- [ ] **패널 토크** (해당 시)
  - [ ] 진행 방식
  - [ ] 사전 질문지 안내
- [ ] **호스트 1:1 콜** - 목적, 일정
- [ ] **당일 안내**
  - [ ] 스피커 사전 모임 시간
  - [ ] 타임라인 테이블

---

## Phase 3: 검토

문서 생성 후 다음을 검토하고, 빠진 항목이 있으면 AskUserQuestion으로 확인하세요:

### 검토 체크리스트

| 항목 | 확인 |
|------|------|
| 밋업 일시가 정확한가? | |
| 장소 정보가 충분한가? (도움 줄 담당자 등) | |
| 발표 시간과 스피커 수가 맞는가? | |
| 준비 일정의 모든 날짜가 채워졌는가? | |
| 타임라인의 시간 계산이 맞는가? | |
| 호스트 연락처가 필요한가? | |
| 스폰서 언급이 필요한가? | |

### 빠뜨리기 쉬운 항목

다음 항목은 자주 빠뜨리니 특별히 확인하세요:

1. **발표 자료 PDF 필수** 안내
2. **공유 가능한 버전** 요청
3. **호스트 랩탑 사용** 안내
4. **발표 사이 질문 없음** 안내
5. **스피커 사전 모임 시간** (밋업 30분 전)
6. **라이브 데모 사전 고지** 요청

## 템플릿

```markdown
# {밋업이름} 스피커 가이드

> 밋업 스피커분들을 위한 안내 문서

---

## 밋업 정보

| 항목 | 내용 |
|------|------|
| 일시 | {날짜} ({요일}) {시간} |
| 장소 | {장소} |
| 형식 | Speaker Talk ({발표시간}분) + Panel Talk |

### 타겟 오디언스
- {오디언스1}
- {오디언스2}

---

## 준비 일정

| 마감일 | 항목 | 설명 |
|--------|------|------|
| ~{1:1콜마감} | 호스트 1:1 콜 | 발표 주제 공유 및 조율 |
| {자료제출일} | 발표 자료 제출 | PDF 또는 공유 링크 |
| {질문전달일} | 사전 질문지 수령 | 패널 토크 예상 질문 전달 |
| {밋업날짜} | 밋업 당일 | {시간} 시작 |

---

## 발표 안내

### 시간
- **{발표시간}분 컷** ({발표시간}분이 되면 호스트가 시간을 안내합니다)
- {스피커수}개의 발표가 연속 진행됩니다
- 발표 간 5분 버퍼가 있습니다 (세팅, 전환 시간)
- **발표 사이에 질문은 받지 않습니다** (패널 토크 후 한번에 진행)

### 발표 주제
자유롭게 정해주시되, 아래 방향을 참고해 주세요:
- {주제방향1}
- {주제방향2}
- {주제방향3}

### 발표 자료
- 제출: **{자료제출일}까지**
- **PDF 버전 필수**: 만일을 위해 반드시 PDF 버전을 추가로 준비해 주세요
- **공유 가능한 버전**으로 만들어 주세요 (참석자들에게 공유 예정)
- 브랜딩 제한 없음: 회사 아이덴티티 팍팍 드러내세요!
- 원활한 진행을 위해 발표는 호스트 랩탑으로 진행합니다

### 라이브 데모
- 라이브 데모가 있다면 미리 알려주세요

---

## 패널 토크

발표 세션 이후, 스피커 전원이 참여하는 패널 토크가 진행됩니다.

### 진행 방식
- 호스트가 사전 준비된 질문으로 대화를 이끕니다
- 패널 토크 후 청중 Q&A 시간이 별도로 있습니다
- 편하게 대화하듯 참여해 주세요

### 사전 질문지
- **{질문전달일}** 에 예상 질문 목록을 전달드립니다
- 사전 질문에 미리 생각을 정리해주시면 좋습니다
- 당일 즉흥 질문도 있을 수 있습니다

---

## 호스트 1:1 콜

밋업 전에 호스트({호스트이름})와 짧은 콜을 진행합니다.

### 목적
- 발표 주제 및 방향 공유
- 패널 토크 질문 방향 조율
- 궁금한 점 해소

### 일정
- **{1:1콜마감}까지** 개별 일정 조율
- 약 15-20분 소요

---

## 당일 안내

### 스피커 사전 모임 ({사전모임시간})

밋업 시작 30분 전에 도착해 주세요. 스피커분들끼리 먼저 인사하고 이야기 나누는 시간입니다.

- 좋은 패널 토크를 위해 서로를 알아가는 시간
- 스피커들끼리 네트워킹도 진짜 밸류가 있습니다
- 평일 저녁이라 마무리 후 시간 내기 어려우실테니, 미리 이야기 나누고 연락처 교환해 주세요

### 타임라인

| 시간 | 내용 | 소요 |
|------|------|------|
| {사전모임시간} | 스피커 도착 및 인사 | 30분 |
| {시작시간} | 밋업 시작 | 10분 |
| {발표1시간} | Speaker Talk 1 | {발표시간}분 |
| {발표2시간} | Speaker Talk 2 | {발표시간}분 |
| ... | ... | ... |
| {휴식시간} | 휴식 | 10분 |
| {패널시간} | Panel Talk | 30분 |
| {QA시간} | 청중 Q&A | 20분 |
| {네트워킹시간} | Networking & Lightning Talks | 30분 |
| {마무리시간} | 마무리 | - |
```
