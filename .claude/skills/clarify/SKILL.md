# Requirement Clarifier

AskUserQuestion 도구를 사용하여 사용자의 요구사항을 단계별로 구체화하는 skill입니다.

## 진행 방식

### Step 1: 목표 파악

먼저 AskUserQuestion으로 물어볼 것:

1. **작업 유형**: 새 기능 개발 / 버그 수정 / 리팩토링 / 문서 작성 / 기타
2. **대상 범위**: 단일 파일 / 여러 파일 / 전체 프로젝트

### Step 2: 세부 요구사항

작업 유형에 따라 추가 질문:

**새 기능 개발인 경우:**
- 어떤 문제를 해결하려는 건가요?
- 비슷한 기존 기능이 있나요?
- 우선순위는? (MVP / 완성도 높게)

**버그 수정인 경우:**
- 재현 방법이 있나요?
- 예상 동작 vs 실제 동작?
- 관련 로그나 에러 메시지?

**리팩토링인 경우:**
- 개선하려는 부분? (성능 / 가독성 / 구조)
- 기존 동작은 유지해야 하나요?

### Step 3: 제약사항 확인

AskUserQuestion으로 확인:

1. **기술적 제약**: 특정 라이브러리 / 패턴 사용 필수?
2. **호환성**: 하위 호환 필요? 특정 환경 지원?
3. **테스트**: 테스트 코드 필요? 기존 테스트 유지?

### Step 4: 요약 및 확인

수집한 정보를 정리하여 보여주고, AskUserQuestion으로 최종 확인:

```markdown
## 요구사항 정리

**목표**: [정리된 목표]
**범위**: [대상 파일/기능]
**세부 요구사항**:
- [항목 1]
- [항목 2]

**제약사항**:
- [제약 1]
- [제약 2]

이대로 진행할까요?
```

## 사용 예시

```
> /clarify
> (Claude가 질문 시작)
> (사용자가 선택/답변)
> (요구사항 정리됨)
> (바로 작업 시작 또는 계획 수립)
```

## 주의사항

- 질문은 최대 4개씩 묶어서 진행 (AskUserQuestion 제한)
- 사용자가 "건너뛰기" 선택 시 해당 항목은 기본값 사용
- 최종 요약 후 바로 작업 시작할지, 계획 먼저 세울지 물어볼 것
