스킬톤이란?
3시간 동안 자신만의 Claude Code 워크플로우를 만들고, 결과물을 공유한 뒤 우승팀을 선정합니다.

요새 해커톤은 엄청 많잖아요. 그런 큰 규모의 해커톤보다는 소규모로 만들어서 찐한 밀도를 만들어보려고 합니다!

이런 고민을 하고 계신가요?

▸ 스킬 설계 패턴
스킬에서 서브에이전트를 부르는 게 맞을까?
아니면 서브에이전트가 스킬을 사용하는 게 더 좋은 패턴일까?

▸ 메타 워크플로우
어제 비슷한 문제를 겪었는데, 오늘도 그 문제를 겪고 있다.
워크플로우를 자동으로 관리할 수는 없을까?

▸ 조직 차원의 Claude Code 활용
우리 팀의 토큰 사용량, 자주 사용하는 스킬,
사용하지 않아서 제거해야 하는 워크플로우는 뭘까?

▸ 개발 외 업무에서의 활용
메일, 슬랙, 노션 같은 툴은 물론, 슬라이드, 비디오 편집까지.
비개발자가 쉽게 적용할 수 있는 방법은 뭘까?

이런 고민을 함께 나누고, 직접 만들어보는 자리입니다.
(이미 쓰고 있는 워크플로우를 스킬톤 시간에 깎으셔도 돼요)
